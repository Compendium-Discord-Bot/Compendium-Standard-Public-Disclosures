# Third-Party License Notices

Most of the open source software we rely on simply asks to be credited, and we credit all of it in [Acknowledgements](Acknowledgements.md "Acknowledgements – Compendium Standard Public Disclosures"). A small number of licenses ask for a little more than a credit. This document lists those cases and explains how we meet each one, so you can see exactly what we owe and how we honor it.

This file is kept in sync automatically. Data for this document is generated using `pnpm export-licenses`, and we re generate it when we add/update/remove a module

## I. License Elections

Some of the software we use comes with a choice of license, letting us decide which set of terms to follow. Where that happens, we pick one license for each package and follow only its terms. Our choices are listed below.

| Package | Offered under | We elect |
| --- | --- | --- |
| `node-forge` | (BSD-3-Clause OR GPL-2.0) | BSD-3-Clause |
| `type-fest` | (MIT OR CC0-1.0) | MIT |

## II. Copyleft Licenses

A few of the packages we use are covered by what are known as copyleft licenses. These licenses can require us to make source code available, but only when the software is distributed to others. We use the packages below while building the product or on our own servers, and we never ship them to you, so that requirement does not apply. Should any of them ever become part of something we distribute, such as the web panel, we will make its source available under its license and keep its notices intact.

| Package | Version(s) | License | Homepage |
| --- | --- | --- | --- |
| `lightningcss` | 1.32.0 | MPL-2.0 | https://github.com/parcel-bundler/lightningcss#readme |
| `lightningcss-win32-x64-msvc` | 1.32.0 | MPL-2.0 | https://github.com/parcel-bundler/lightningcss#readme |
